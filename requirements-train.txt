# Training dependencies for HF Jobs runs.
# Base image must provide torch + CUDA (e.g. pytorch/pytorch:2.5.1-cuda12.4-cudnn9-runtime).
#
# Stack (decided after iterating on HF Jobs):
#   - HF transformers + peft + bitsandbytes  (model load, 4-bit QLoRA, LoRA wrap)
#   - HF TRL SFTTrainer                       (SFT phase)
#   - In-tree GRPO/PPO/REINFORCE              (RL phase; TRL's GRPOTrainer OOMs on T4)
#
# Unsloth was tried but its dep tree forced peft>=0.18, which has a runtime
# import bug against transformers >=4.50 (`BloomPreTrainedModel` no longer
# at top level). We don't need Unsloth's 2x speed/lower-VRAM benefit on a 0.5B
# model running on 24 GB+ GPUs, so it's not worth blocking the whole pipeline.
#
# These pins are exactly the combination the Kaggle/Colab notebook ran
# successfully on T4 (verified end-to-end).
transformers>=4.46,<5.0
peft>=0.13,<0.18
trl>=0.13,<0.16
accelerate>=0.34
bitsandbytes>=0.43,<0.50
huggingface_hub>=0.25,<2.0
sentencepiece
einops
datasets<4.0

pydantic>=2.5
